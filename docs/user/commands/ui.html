Command: ui, mousemode

The ui command sets generic tool-interface preferences and adjusts mouse-button assignments for interactive manipulation.

• ui autostart  true | false  tool-name

Whether the tool should start automatically upon subsequent ChimeraX startup; saved in preferences. Can also be adjusted through context menus.

• ui dockable  true | false  tool-name

Whether panels of the tool should be allowed to reinsert automatically when dragged over the main window, as opposed to remaining separate; initially true for all panels except the Help Viewer; saved in preferences. Can also be adjusted through context menus.

• mousemode [ control ] [ alt ] [ command ] [ shift ] [ leftMode  function ] [ middleMode  function ] [ rightMode  function ] [ wheelMode  function ] [ pauseMode  function ]

The mousemode command (same as ui mousemode) assigns ChimeraX functions to the left, middle, and right mouse buttons, alone and in combination with the modifier keys control, alt, command, and shift. In addition, wheelMode refers to scrolling and pauseMode refers to pausing the cursor over something in the graphics window without clicking any buttons.

A touchpad or single-button mouse can also be used. See also: device, Mouse Modes for Right Button toolbar, Marker Placement, ChimeraX window

Given without arguments, the mousemode command reports the current assignments in the Log. Initial defaults include:

  • left: rotate
  • middle: translate
  • right: translate
  • wheel (scrolling): zoom
  • Ctrl-left: new selection
  • Shift-Ctrl-left: toggle selection

Possible values of function are listed in the table below. They can be truncated to unique strings. If function contains spaces, it should be enclosed in quotation marks. Not all functions make sense for a given input mode; for example, only identify object and none are reasonable functions for pauseMode.

Multiple modifier keys and multiple “buttons” can be given in a single command, but all specified modifiers apply to all specified buttons. For example, two commands rather than one would be needed to assign functions to control right and alt wheel:

mousemode control right 'rotate and select'
mousemode alt wheel contour

← Touchpad or Single-Button Mouse

A click with a touchpad (trackpad) or single-button mouse is generally interpreted as left click, but can be combined with modifier keys to emulate middle and right:

  • On a Mac, the modifier keys to emulate middle and right are alt and command (), respectively. In other words, leaving alt left and command left unassigned allows them to serve the middle- and right-button functions. Multitouch actions can also be used for manipulation, if enabled in the Trackpad preferences; if multitouch is turned off, a two-finger drag is interpreted as scrolling (wheel, default assignment zoom). Regardless of the multitouch setting, if “secondary click” is enabled in the Trackpad system preferences, a two-finger click-and-drag performs the right-button action.

    Magic Mouse issue: if a trackpad is also present, scrolling with the Magic Mouse will not work unless multitouch actions are disabled in the Trackpad preferences (the Trackpad gestures option is unchecked).

  • On Windows, the modifier keys to emulate middle and right are control alt and alt, respectively. In addition, on many Windows 10 laptops, a click in the lower right corner of the trackpad is a right click, and right-click-and-drag can be done by clicking in the lower right corner with one finger and dragging with another finger near the middle of the trackpad. On laptops with a “precision touchpad,” a two-finger tap is also interpreted as right-click, but a two-finger drag still performs scrolling.
