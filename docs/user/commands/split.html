Command: split

Usage:
split  model-spec  [ chains ] [ ligands ] [ connected ] [ atoms  atom-spec ]n

The split command partitions atomic models into separate submodels in the hierarchy. For example, model #1 could be split into #1.1, #1.2, etc. Splitting parts of a model into separate submodels may facilitate their independent control, such as with Model Panel checkboxes or mouse modes that act on whole models. See also: combine

As per the usual convention, a blank specification means “all.” That is, if no models are specified, all atomic models will be affected. There is no way to undo this operation except by reopening the original data.

How to split can be indicated with one or more of the following:

  • chains - by chain ID (default)
  • ligands - each ligand in a separate submodel, the remaining atoms in another
  • connected - into covalently bonded sets of atoms
  • atoms  atom-spec  - can be used multiple times in the same command, for example:
    split #1 atoms :1-100 atoms :101-150 atoms :151-200
    ... to place each set of specified atoms, as well as the set of unspecified atoms, in a separate submodel.
If none of the above are given, the chains method will be used. If there is nothing to split, the original model will be left unaltered.
