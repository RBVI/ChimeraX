Command-Line Color Specification

Colors can be specified in commands (color, etc.) with:

  • the color names shown below
  • color names defined by the user with color name
  • hexadecimal format starting with # and encoding rgb (3, 6, or 12 digits) or rgba (4, 8, or 16 digits), for example: #00ced1
  • rgb or rgba given as three or four comma-separated values in the range 0-100 (no spaces, no % signs), for example: 75,10,55
  • CSS-like specifications in the following formats:
    • rgb(red, green, blue)
    • rgba(red, green, blue, alpha)
    • hsl(hue, saturation, lightness)
    • hsla(hue, saturation, lightness, alpha)
    • gray(grayscale)
    The colorspace name must be in lowercase and immediately precede the open parenthesis. Otherwise, embedded spaces are allowed. Values for red, green, blue, alpha (increasing transparent → opaque), saturation, lightness, and grayscale (increasing black → white) can be given as:
    • percentages indicated with %
    • fractional values in the range 0-1, indicated with a decimal point (for example, 1. rather than 1)
    The red, green, blue, and grayscale components can also be expressed as whole numbers in the range 0-255. Values for hue are in the range 0-360, corresponding to angular positions on the color wheel. See more examples.

The command color list all lists the available names along with color swatches in the Log. Color names can be truncated, with ambiguous truncations going to the shortest of the matching full names. For the color command, however, color names starting with “pale” should be expanded sufficiently to distinguish them from the palette keyword, and enclosed in quotation marks or with space omitted. See also: initial colors, element colors
