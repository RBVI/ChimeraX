Command: minimize

Usage:
minimize  atomic-model-spec  [ dockPrep  true | false  [ dockprep-options ]] [ maxSteps  N ] [ liveUpdates  true | false ] [ logEnergy  true | false ]

**In progress** and currently works only for structures containing only “standard” residues as defined in the Amber14 parameter files. These include standard amino and nucleic acid residues, water, and several lipids and metal ions, but not arbitrary ligands. Only a single model should be specified. The minimization is carried out using the OpenMM LocalEnergyMinimizer with the LangevinIntegrator.

By default (dockprep true), the minimize command calls dockprep to perform several tasks to prepare the structure beforehand, such as:

  • deleting water molecules
  • repairing truncated sidechains
  • adding hydrogens
  • assigning partial charges

The dockprep command options can also be used in the minimize command to control these processes.

The maxSteps option sets the number of iterations (default 0, meaning that minimization will continue until convergence regardless of how many iterations it takes). Convergence is defined as when the root-mean-square of all force components falls to 10 or lower.

The liveUpdates option (default true) specifies whether to also update the structure when the energy is reported in the status line, currently every 100 steps.

The logEnergy option indicates whether to report the energy in the Log (default false, energy reported in the status line only).
