The conference command shares a session between two or more ChimeraX instances running on different computers. Its main purpose is to allow multi-person virtual reality (VR) sessions in ChimeraX, although it can also be used without VR. Each participant is represented by an image and hand-controller pointers (or if not using VR, the mouse pointer) so that participants can see each other pointing at objects in the scene. It is also implemented as the Conference Call tool.

A “conference call” allows using a publicly accessible address/port to run the hub server, whereas the related command meeting only allows running the hub server on the initiator's local host, which may not be reachable through participants' firewalls. Other limitations, however, may be similar. See also: vr

The location specifies the host (hostname or IP address), port number, and conference-call name in the form host:port/conference_call_name. If the port is omitted (no colon), then port 443 will be used. If the conference call name is omitted (no slash), then unnamed will be used. The hub server hosted by the UCSF RBVI is: cxconference.rbvi.ucsf.edu:443.

The action can be:

  • start – start a conference call on the hub server; location must specify the host and a name for the conference call
  • join – join a conference call; location must specify the host, and unless the hub server was started locally with the host action (below), the name of the conference call
  • host – start a hub server on the local host (equivalent to using meeting start); since only a single session is supported in this scenario, location can be omitted

Aside from synchronizing pointers, the conference command will copy a session from the ChimeraX instance that initiates the conference call to each person who joins. This copying occurs only when a person joins. Commands executed by one participant will automatically execute for the others (by default), and if a participant using VR moves the scene, others using VR will see the motion. Participants not using VR maintain their own viewing directions. If unshared changes have been made, a participant can still share his current scene with the others by using conference send.

The settings of an existing (already started) conference call can be changed with conference set.

Participants can exit the shared session with conference close. If the person who started the conference exits, one of the remaining attendees should issue the command conference set with copyScene true if more attendees are expected to join.

Options

name  participant-name
Name to show for a participant's pointer and image models in the Model Panel.
color  pointer-color
Color for the cones representing a VR participant's hand-controller pointers (or non-VR participant's mouse pointer).
faceImage  image-file
Image to represent a VR participant's headset position, where image-file is the pathname of a JPG or PNG file. The pathname can be absolute or relative to the current working directory as reported by pwd, Substituting the word browse for the pathname brings up a file browser window for choosing the file interactively.
copyScene  true | false
Whether to copy the session from the starting instance to each person who joins (default true). This copying is done only when a person joins the shared session.
relayCommands  true | false
Whether to share commands with other participants, and in turn, execute other participants' commands automatically (default true). Note that toolbar icon buttons and many mouseclick/pointer operations act via commands.
updateInterval  M
How often to send a participant's VR headset and hand-controller positions to the other participants (every Mth frame, default 1, every frame). For non-VR users, the mouse pointer is not tracked continuously; the position is only displayed to others when it hovers over an atom.
