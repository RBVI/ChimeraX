Tool: Conference Call

Warning: not yet working, use meeting instead

Conference Call allows sharing a session between two or more ChimeraX instances running on different computers. Its main purpose is to allow multi-person virtual reality (VR) sessions in ChimeraX, although it can also be used without VR. Each participant is represented by an image and hand-controller pointers (or if not using VR, the mouse pointer) so that participants can see each other pointing at objects in the scene. The command-line implementation is conference.

A “conference call” allows using a publicly accessible address/port to run the hub server, whereas the related command meeting only allows running the hub server on the initiator's local host, which may not be reachable through participants' firewalls. Other limitations, however, may be similar. See also: vr

Conference Call can be opened from the Utilities section of the Tools menu and manipulated like other panels (more...). Clicking Parameters or Log controls which of the mutually exclusive sections is expanded in the dialog; Parameters are described below, whereas the Log is simply a log of conference events.

The hub server provided by the UCSF RBVI can be specified as host cxconference.rbvi.ucsf.edu and port 443.

Conference parameters:

  • Host – hostname or IP address for hub server, required to start and join
  • Port – port number
  • Conference name – a name for the conference, required to start (and join, unless the conference was started with the Host button below)
  • User name – name to show for a participant's pointer and image models in the Model Panel

Buttons:

  • Start – start a conference call on the hub server
  • Join – join a conference call
  • Host – start a hub server on the local host (equivalent to using meeting start)
  • Broadcast Session – share current session with others, e.g. after making any unshared changes (conference send)
  • Leave – exit the shared session (conference close)
