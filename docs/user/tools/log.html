Tool: Log

The Log tool echoes commands that have been executed, with any truncations expanded. It also shows messages and other output information, potentially including hyperlinks and command-execution links (for example, to generate an assembly or to show the sequence of a chain). Restoring a session restores its Log contents as a collapsible section with a disclosure triangle. See also: Command History, ChimeraX browser, log, usage, model information, measurements

Like other tools, the Log can be opened from the Tools menu (more...). Its context menu includes:

  • Clear – remove all contents
  • Copy Selection
  • Executable Command Links – whether command links should be executable or show help (saved as a preference)
  • Insert Image – include a thumbnail image of the current display
  • Notifications (settings saved as preferences)
    • Errors Shown in Dialog – whether error messages in the Log should also generate a separate dialog window
    • Raise Log When Logging Occurs – whether to show the Log (bring to front if tabbed behind another panel, or reopen if closed) whenever some action or message adds to its contents
    • Warnings Shown in Dialog – whether warning messages in the Log should also generate a separate dialog window
  • Restoring Session Clears Dialog – whether to clear the current contents from the Log before populating it with the contents from a session that is being restored (saved as a preference)
  • Select All
  • Save As... save contents to an HTML file
